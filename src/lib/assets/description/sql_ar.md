### **ملخص سريع (TL;DR)**
لغة **SQL** (لغة الاستعلام الهيكلية) هي اللغة القياسية المتخصصة للتواصل مع **أنظمة إدارة قواعد البيانات (RDBMS)**. تتيح لك هذه اللغة إنشاء البيانات، واسترجاعها، وتحديثها، وحذفها (عمليات CRUD) من جداول مهيكلة مرتبطة ببعضها البعض عبر علاقات محددة.

## **لغة البيانات**
تنفرد SQL بكونها لغة **تصريحية (Declarative)**؛ فأنت لا تخبر الحاسوب *كيف* يجد البيانات (كما تفعل في الحلقات التكرارية - loops)، بل تخبره *ما هي* البيانات التي تريدها، ويتولى محرك قاعدة البيانات عملية التحسين (Optimization).

### **العمليات الأساسية**
* **تعريف البيانات (DDL):** تُستخدم لتحديد الهيكل (Schema)، مثل `CREATE TABLE` أو `ALTER TABLE`.
* **التعامل مع البيانات (DML):** تُستخدم لإدارة البيانات نفسها، مثل `SELECT` و `INSERT` و `UPDATE` و `DELETE`.
* **العلاقات:** ربط الجداول باستخدام **المفاتيح الأساسية (Primary Keys)** (معرفات فريدة) و **المفاتيح الأجنبية (Foreign Keys)** (إشارات إلى جداول أخرى).
* **عمليات الربط (Joins):** هي "القوة الخارقة" للغة SQL، حيث تسمح لك بدمج البيانات من جداول متعددة بناءً على أعمدة مرتبطة.

> فكر في قاعدة البيانات كأنها **مكتبة ضخمة**:
> * **SQL** هي **أمين المكتبة**.
> * **الجداول** هي **أرفف الكتب**، المنظمة حسب الفئة (مثل "المستخدمين"، "الطلبات").
> * **الصف (Row)** هو **كتاب محدد** على ذلك الرف.
> * **العمود (Column)** هو **تفصيل محدد** عن ذلك الكتاب (العنوان، المؤلف، الرقم المعياري).
> إذا أردت العثور على كل كتاب ألفه كاتب معين في عام 2026، فلن تتجول بين الممرات بنفسك؛ بل تقدم طلباً محدداً لأمين المكتبة (استعلام `SELECT`) وهو يحضر الكتب إلى مكتبك.

---

### **مقتطف برميجي (Snippet)**

```sql
-- إنشاء جدول بسيط
CREATE TABLE artists (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    specialty VARCHAR(50)
);

-- استعلام باستخدام JOIN لمعرفة أسماء الفنانين وعناوين أعمالهم
SELECT artists.name, artworks.title
FROM artists
JOIN artworks ON artists.id = artworks.artist_id
WHERE artists.specialty = 'Digital Art'
ORDER BY artworks.created_at DESC;
```
