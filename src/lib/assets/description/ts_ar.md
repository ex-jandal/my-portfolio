### **ملخص سريع (TL;DR)**
لغة **TypeScript** هي مجموعة عليا (Superset) قوية من JavaScript تضيف ميزة **التنويع الثابت (Static Typing)**. تعمل كشبكة أمان لاصطياد الأخطاء البرمجية أثناء عملية التطوير (وقت التحميل) بدلاً من تركها تتسبب في انهيار تطبيقك أمام المستخدمين (وقت التشغيل).

---

## **لماذا TypeScript؟**
بحلول عام 2026، أصبحت TypeScript هي المعيار الصناعي لتطوير الويب الاحترافي. فهي تأخذ مرونة JavaScript وتضيف إليها **قابلية التنبؤ** التي تتميز بها اللغات المعتمدة على الأنواع (Typed languages).

### **المزايا الرئيسية**
* **سلامة الأنواع (Type Safety):** حدد بالضبط شكل بياناتك (نصوص، أرقام، كائنات مخصصة). إذا حاولت تمرير نوع خاطئ، فلن يتم تحويل الكود (Compile).
* **إكمال تلقائي فائق:** بما أن المحرر يعرف بنية بياناتك، فإنه يقدم اقتراحات دقيقة، مما يقلل حاجتك للتحقق المستمر من التوثيق (Documentation).
* **ثقة عند إعادة الهيكلة (Refactoring):** هل تحتاج إلى تغيير اسم خاصية في 50 ملفاً؟ ستقوم TypeScript بالعثور على كل مرجع وتحديثه، أو تنبيهك في حال حدوث كسر في الكود.
* **نظام بيئي حديث:** أطر العمل مثل **NestJS** مبنية من الصفر للاستفادة من قوة TypeScript.


> البرمجة بلغة JavaScript تشبه قيادة سيارة في مدينة جديدة بدون خريطة؛ يمكنك الذهاب إلى أي مكان، لكنك غالباً ستتخذ منعطفات خاطئة. أما البرمجة بلغة **TypeScript** فهي تشبه وجود **نظام GPS** مع تحديثات مباشرة لحركة المرور؛ لن يمنعك من القيادة، لكنه سيصرخ في وجهك بمجرد أن تحاول الدخول في شارع ذو اتجاه واحد!

### **مقتطف برمجي (Snippet)**
لاحظ كيف تمنع TypeScript خطأً حسابياً شائعاً قد يمر بصمت في JavaScript.

```typescript
interface User {
    name: string;
    id: number;
}

function welcomeUser(user: User) {
    console.log(`أهلاً يا ${user.name}! المعرف الخاص بك هو: ${user.id}`);
}

const abu = { name: "Abu_Jandal", id: "123" }; 

// ❌ خطأ من TypeScript:
// الوسيط من نوع '{ name: string; id: string; }' لا يمكن تعيينه للبارامتر من نوع 'User'.
// أنواع الخاصية 'id' غير متوافقة. النوع 'string' لا يمكن تعيينه للنوع 'number'.
welcomeUser(abu); 
```
