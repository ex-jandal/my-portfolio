### **ملخص سريع (TL;DR)**
إطار **Blazor** هو إطار عمل للويب من Microsoft يتيح لك بناء واجهات مستخدم تفاعلية باستخدام لغة **C#** بدلاً من JavaScript. كونه جزءاً من نظام .NET البيئي، فهو يُمكّن المطورين من مشاركة الكود والمكتبات بين الخادم (Server) والعميل (Client)، مما يجعل مفهوم "Full-stack C#" حقيقة ملموسة.

---

## **كيف يعمل Blazor؟**
تكمن "بِراعة" Blazor في قدرته على تشغيل كود C# مباشرة داخل المتصفح، وذلك عبر نموذجين للاستضافة:

### **1. Blazor WebAssembly (WASM)**
يتم تحميل بيئة تشغيل .NET بالكامل إلى المتصفح باستخدام تقنية **WebAssembly**. يعمل كود C# الخاص بك من جهة العميل تماماً كما يفعل JavaScript.
* **دعم العمل دون اتصال:** بمجرد التحميل، يمكن للتطبيق العمل بدون اتصال نشط بالإنترنت.
* **لا يوجد عبء على الخادم:** المتصفح هو من يتولى كل العمليات الشاقة.

### **2. Blazor Server**
يعمل التطبيق على الخادم. عندما يتفاعل المستخدم مع الصفحة، يتم إرسال رسالة صغيرة عبر اتصال **SignalR (WebSockets)**؛ حيث يقوم الخادم بحساب التغيير في واجهة المستخدم ويرسل فقط "الفروقات" (diff) لتحديث الصفحة.
* **تحميل أولي سريع:** يتم تحميل سكريبت صغير فقط في البداية.
* **أمان عالٍ:** الكود الخاص بك لا يغادر الخادم أبداً، مما يسهل حماية الملكية الفكرية.

> تخيل أنك في مطعم بدولة أجنبية:
> * **Blazor Server** يشبه وجود **مترجم** معك على الهاتف؛ في كل مرة تريد طلب شيء، تتصل به، فيترجم ويخبر النادل نيابة عنك.
> * **Blazor WebAssembly** يشبه **تعلم اللغة** قبل الذهاب؛ يستغرق الأمر وقتاً للدراسة (التحميل الأولي)، ولكن بمجرد وصولك، يمكنك التحدث مباشرة مع النادل دون الحاجة للاتصال بأحد!

### **مقتطف برمجي (Snippet)**
يستخدم Blazor ملفات بتنسيق `.razor` تمزج بين HTML وكود C# بسلاسة تامة.

```razor
@page "/counter"

<h1>عداد النقرات</h1>

<p role="status">العدد الحالي: @currentCount</p>

<button class="btn btn-primary" @onclick="IncrementCount">اضغط هنا</button>

@code {
    private int currentCount = 0;

    private void IncrementCount()
    {
        currentCount++;
    }
}
```
