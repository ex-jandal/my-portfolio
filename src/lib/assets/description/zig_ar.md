### **ملخص سريع (TL;DR)**
**Zig 0.16** هي لغة أنظمة حديثة متعددة الأغراض وبيئة أدوات (toolchain) صُممت من أجل **المتانة، الكفاءة القصوى، وقابلية الصيانة**. تهدف لتكون "نسخة أفضل من C"، حيث تلغي تدفق التحكم الخفي (hidden control flow) وتخصيصات الذاكرة المستترة، مع توفير نظام بناء (build system) قوي يغني عن استخدام Make و CMake.

## **لماذا لغة Zig؟**
تنفرد Zig بعدم وجود **تدفق تحكم خفي** (لا توجد تخصيصات ذاكرة مخفية، ولا استدعاءات دالة مستترة) ولا تعتمد على **المعالج المسبق (preprocessor)**. ما تراه في الكود هو بالضبط ما يحدث عند التنفيذ.

### **الركائز الأساسية**
* **وقت التحميل (Comptime):** تستخدم Zig ميزة تنفيذ الكود أثناء وقت التحميل للتعامل مع العموميات (generics) والماكرو. إذا كان بإمكانك كتابته بلغة Zig، يمكنك تشغيله في وقت التحميل.
* **ذاكرة يدوية مع شبكة أمان:** على عكس C، تشجع Zig على استخدام "المخصصات" (Allocators) بشكل صريح، مما يجعل مكان وكيفية استخدام الذاكرة واضحاً تماماً.
* **التكامل مع C:** لغة Zig هي أيضاً مترجم لغة C. يمكنك إضافتها إلى مشروع C، ويمكنها ترجمة أكواد C/C++ مباشرة.
* **بدون بيئة تشغيل (No Runtime):** لا يوجد ثقل "مكتبة قياسية" مخفية أو مجمع نفايات. إنه كود آلة نقي.

> فكر في Zig مثل **ساعة ميكانيكية**؛ كل ترس ونابض فيها مرئي للعين. لا توجد أجزاء "سحرية" مخفية خلف الغطاء؛ إذا تحرك شيء ما، يمكنك رؤية السبب والكيفية بدقة.

### **مقتطف برمجي (Snippet)**
في إصدار Zig 0.16، تتطلب منك المهام -مثل الحصول على مسار المجلد الحالي أو تخصيص الذاكرة- أن تكون صريحاً بشأن "المخصص" (Allocator) المستخدم.

```zig
const std = @import("std");

pub fn main(init: std.process.Init) !void {
    const io = init.io;
    const gpa = init.gpa;

    var working_dir = std.Io.Dir.cwd();
    
    const content = 
        try working_dir.readFileAlloc(io, "config.json", gpa, .unlimited);
    defer gpa.free(content);

    const stdout = std.Io.File.stdout().writer(io, &.{});
    try stdout.interface.print("المحتوى: {s}\n", .{content});
}
```
