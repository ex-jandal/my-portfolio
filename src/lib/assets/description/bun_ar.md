### **ملخص سريع (TL;DR)**
**Bun** هي بيئة تشغيل **JavaScript** فائقة السرعة ومتكاملة (All-in-one)، تعمل أيضاً كمدير حزم ومجمع للأكواد (Bundler). كُتبت Bun من الصفر باستخدام لغة **Zig** لتكون بديلاً لـ Node.js، مع التركيز على الأداء الأقصى، والدعم الأصلي لـ TypeScript، وتوحيد تجربة التطوير.

## **أداة "كل شيء"**
صُممت Bun للقضاء على "إرهاق أدوات التطوير" (Toolchain fatigue) في ويب الحديث. بدلاً من تثبيت خمس أدوات مختلفة للاختبار والتجميع وتشغيل الكود، يمكنك ببساطة استخدام أمر `bun` فقط.

### **الميزات الرئيسية**
* **محرك JavaScriptCore:** على عكس Node.js وDeno (اللذين يستخدمان محرك V8 من جوجل)، تعتمد Bun على محرك **JavaScriptCore** (المطور لمتصفح Safari)، والذي يمتاز بسرعة بدء تشغيل أعلى واستهلاك أقل للذاكرة.
* **دعم أصلي لـ TypeScript وJSX:** لا حاجة لـ `tsc` أو `babel`؛ فـ Bun تشغل ملفات `.ts` و `.tsx` و `.jsx` بشكل أصلي ومباشر.
* **سرعة فائقة:** سواء كان الأمر يتعلق بتثبيت الحزم (`bun install`) أو تشغيل خادم التطوير، غالباً ما تكون Bun أسرع بـ **10 إلى 100 مرة** من npm أو Node.
* **واجهات برمجة (APIs) مدمجة:** تتضمن Bun أدوات عالية الأداء لخوادم HTTP، وإدخال/إخراج الملفات، وقواعد بيانات SQLite، مع توافق كامل مع معظم وحدات Node.js المدمجة.

> تخيل Node.js كأنها **ورشة عمل** تضطر فيها لشراء المثقاب والمشار والمطرقة بشكل منفصل (npm, vite, jest، إلخ). أما **Bun فهي السكين السويسري**؛ تخرجها من جيبك لتجد كل أداة تحتاجها متصلة بمقبض واحد. وبما أنها مكتوبة بلغة **Zig**، فاعتبر أن هذه السكين مصنوعة من تيتانيوم عالي الجودة!

### **مقتطف برمجي (Snippet)**
إعداد خادم في Bun بسيط ومختصر للغاية مقارنة بـ Node.js التقليدي.

```javascript
// server.ts - `bun server.ts` قم بتشغيل الملف مباشرة عبر أمر 
const server = Bun.serve({
  port: 3000,
  fetch(request) {
    const url = new URL(request.url);
    if (url.pathname === "/") return new Response("أهلاً يا Abu_Jandal!");
    return new Response("404 - غير موجود!");
  },
});

console.log(`الخادم يعمل على: http://localhost:${server.port}`);
```
