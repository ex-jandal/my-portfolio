### **ملخص سريع (TL;DR)**
نظام **Git** هو نظام **تحكم في الإصدارات (VCS)** موزع، يقوم بتتبع التغييرات في الكود المصدري أثناء عملية تطوير البرمجيات. يتيح Git لعدة مطورين العمل على نفس المشروع في وقت واحد دون أن يمسح أحدهم عمل الآخر، كما يوفر سجلاً كاملاً لكل تعديل تم إجراؤه على الإطلاق.

## **آلة الزمن الخاصة بالكود**
تكمن أهمية Git في أنه يسمح لك بـ "حفظ" مشروعك في نقاط زمنية محددة. إذا ارتكبت خطأً أو تسببت ميزة جديدة في تعطل كل شيء، يمكنك ببساطة العودة إلى حالة سابقة مستقرة.

### **الركائز الأساسية**
* **بنية موزعة:** يمتلك كل مطور نسخة كاملة من سجل المشروع على جهازه الخاص، وليس فقط على خادم مركزي. هذا يجعل Git سريعاً للغاية وموثوقاً.
* **الحالات الثلاث:** يتعامل Git مع الملفات في ثلاث مناطق رئيسية: **مجلد العمل (Working Directory)** (التعديلات الحالية)، **منطقة التحضير (Staging Area)** (التجهيز للحفظ)، و**المستودع (Repository)** (السجل الدائم).
* **التفرع والدمج (Branching & Merging):** يمكنك إنشاء "فروع" للعمل على ميزات جديدة بمعزل عن الكود الأساسي. وبمجرد جاهزية الميزة، تقوم بـ "دمجها" مرة أخرى في الكود الرئيسي.
* **نزاهة البيانات:** يتم تشفير كل تغيير باستخدام خوارزمية (SHA-1)؛ ومن المستحيل تقريباً تغيير ملف أو مجلد دون أن يكتشف Git ذلك.

> فكر في Git كأنها **لعبة RPG صعبة**:
> * **`git add`**: تشبه الوقوف أمام باب "الوحش"؛ فأنت تختار الأدوات (الملفات) التي تريد أخذها معك للمعركة.
> * **`git commit`**: هي **نقطة الحفظ (Save Point)**؛ إذا خسرت المعركة (تعطل الكود)، فلن تضطر لإعادة اللعبة من البداية، بل ستقوم فقط بتحميل آخر نقطة حفظ.
> * **`git push`**: تشبه رفع ملف الحفظ الخاص بك إلى "السحابة" حتى يتمكن أصدقاؤك من رؤية تقدمك أو مساعدتك.
